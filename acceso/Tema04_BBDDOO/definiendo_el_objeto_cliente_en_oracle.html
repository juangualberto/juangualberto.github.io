Definiendo el objeto Cliente en Oracle

Definir tipos en Oracle es muy similar a definir tablas, incluso se usan los mismos tipos base, con la diferencia que también se podríamos definir métodos. Veámoslo con un ejemplo.

Definiendo el objeto cliente en SQL99

Para definir nuestro objeto "Cliente" vamos a usar los tipos base (VARCHAR, NUMBER, INTEGER...) y tipos compuestos que vamos a crear previamente.

En primer lugar vamos a definir el tipo "ListaTelefonos" para albergar la lista de teléfonos de nuestros clientes:

CREATE TYPE ListaTelefonos_vartyp AS VARRAY(10) OF VARCHAR2(20);
/

Seguidamente creamos el "Tipo dirección", para albergar la información referente a este dato de cada cliente.

CREATE TYPE Direccion_objtyp AS OBJECT (
	Calle		VARCHAR2(200),
	Ciudad		VARCHAR2(200),
	Comunidad	VARCHAR2(2),
	CodPostal	VARCHAR2(10)
);

Finalmente podemos crear el "Tipo Cliente", para el que usamos tanto tipos base como los tipos de datos que acabamos de crear:


CREATE TYPE Cliente_objtyp AS OBJECT (
	Cliente			NUMBER,
	NombCliente		VARCHAR2(200),
	Direccion_obj		Direccion_objtyp,
	ListaTelefonos_var	ListaTelefonos_vartyp,

	ORDER MEMBER FUNCTION
		ComparaPedidosCliente (x IN Cliente_objtyp) RETURN INTEGER
);
/

Es importante darnos cuenta que este modelo presenta importantes ventajas. Por ejemplo la lista de teléfonos equivale a una tabla anidada dentro del cliente.

Cabe citar, que podemos modificar un tipo a posteriori exactamente igual que alteramos la estructura de una tabla en un sistema relacional clásico:

CREATE OR REPLACE TYPE BODY Cliente_objtyp AS
ORDER MEMBER FUNCTION
	ComparaCliente (x IN Cliente_objtyp) RETURN INTEGER IS
	BEGIN
		RETURN Cliente - x.Cliente;
	END;
END;
/
