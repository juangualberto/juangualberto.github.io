Espacios de nombres

Los espacios de nombres tienen dos propósitos:

  1. Distinguir entre elementos y atributos de distintos vocabularios con diferentes significados que puedan compartir el mismo nombre.
  2. Agrupar todos los elementos y atributos relacionados de una aplicación XML concreta para que el software pueda fácilmente reconocerlos.

Para evitar las colisiones de nombres se declaran espacios de nombres mediante el atributo reservado xmlns (xml namespace) con la sintaxis xmlns:prefijo="URI".
Una URI (Uniform Resource Identifier) es una cadena de caracteres que identifica un recurso.
Por ejemplo para XSLT (Extensible Stylesheet Language Transformations) el prefijo xsl identifica la URI donde está definido el espacio de nombres XSLT:
xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
Una vez definido el namespace podemos referirnos a cada elemento sin ambigüedad usando su nombre cualificado (Qualified Name o Qname) mediante la notación Prefijo:ParteLocal (en inglés Prefix:LocalPart). Un QName para el namespace XSLT es: xsl:template.
Otro ejemplo de especificación de espacios de nombre por defecto html para simplificar la escritura XHTML sería:
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="es" lang="es">
Así no hace falta el prefijo y los dos puntos para los elementos como <body> o </head> en lugar de <html:body> o </html:head>.
Siguiendo con el ejemplo_CDATA.xhtml si eliminamos la sección CDATA no hay confusión entre las etiquetas title que aún teniendo el mismo nombre pertenecen a namespaces diferentes. El título de la página XHTML no se confunde con el título del tooltip para el gráfico svg.
El espacio de nombres para el gráfico svg también se ha definido por defecto, pero solamente afecta el namespace a la zona que termina en </svg>.
<svg xmlns="http://www.w3.org/2000/svg">

Si se hubiera definido sin usar la opción por defecto, se haría de la siguiente forma:
    <svg:svg xmlns:svg="http://www.w3.org/2000/svg">
        <svg:title>Círculo Azul</svg:title>
        <svg:circle cx="3cm" cy="2cm" r="30" fill="blue" />
    </svg:svg>
En ese caso todos los elementos deben nombrarse mediante el QName como <svg:title> o </svg:svg>.
